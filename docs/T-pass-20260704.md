# T pass — transition bug fixes + 5 new states (2026-07-04)

Preston previewed at localhost and flagged: (1) a "double morph" on the state
before a triangle, (2) an intermittent blank white screen, (3) the missing
ornament designs he wanted as cycle states. Diagnosed with the harness, fixed all
three. Evidence: `scratchpad/visual-matrix/` (13-state × 8-theme grid), the blank
scan (0 blank / 0 thin frames over 340 live frames).

## Root cause of both transition bugs — clip-path
Both were clip-path-exclusive (the only feature using it, on the triangle state):
- **Double morph**: arming the clip snapped the shape from its CSS border-radius to
  a matched-vertex polygon that *undershot* the radius (quarters clipped to 74–86%
  of its box) — a visible second "morph into itself."
- **Blank screen**: framer interpolating a rect polygon → a triangle polygon (one
  corner collapsed onto the opposite side) **self-intersected mid-tween** → shapes
  clipped to ~0 area (measured `[0.03, 0.006, 0, 0, 0, 0]`) → white screen.

## Fix — remove clip-path entirely
Every silhouette now morphs via **border-radius + rotate + transform** only (rock-
solid, no degeneracy). The literal-triangle state is retired (triangles *require*
clip-path). Angular/sharp energy now comes from **rotated squares (diamonds) + 0-
radius corners**, which morph flawlessly — and which are exactly what the new
designs use. Result: **0 blank / 0 thin frames across all 13 states + transitions.**
